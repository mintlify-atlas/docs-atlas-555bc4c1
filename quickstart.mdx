This guide will walk you through installing the Prompt Improver and trying it with your first prompts.

<Steps>
  <Step title="Install via marketplace">
    Add the severity1 marketplace and install the plugin:

    ```bash
    claude plugin marketplace add severity1/severity1-marketplace
    claude plugin install prompt-improver@severity1-marketplace
    ```

    Then restart Claude Code.
  </Step>

  <Step title="Verify installation">
    Confirm the plugin is active:

    ```bash
    claude /plugin
    ```

    You should see `prompt-improver` in the list of installed plugins.
  </Step>

  <Step title="Try a vague prompt">
    Submit a vague prompt to see the plugin in action:

    ```bash
    claude "fix the error"
    ```

    **What happens:**

    Claude will evaluate your prompt and determine it's vague. You'll see a friendly notification:

    ```
    Hey! The Prompt Improver Hook flagged your prompt as a bit vague because 
    it's unclear which error you're referring to.
    ```

    Then Claude will research your project and ask targeted questions:

    ```
    Which error needs fixing?
      ○ TypeError in src/components/Map.tsx (recent change)
      ○ API timeout in src/services/osmService.ts
      ○ Other (paste error message)
    ```

    Select an option, and Claude will proceed with full context.
  </Step>

  <Step title="Try a clear prompt">
    Now submit a clear, specific prompt:

    ```bash
    claude "Fix TypeError in src/components/Map.tsx line 127 where mapboxgl.Map constructor is missing container option"
    ```

    **What happens:**

    Claude evaluates the prompt, determines it's already clear, and proceeds immediately without asking any questions. You get zero skill overhead for clear prompts.
  </Step>
</Steps>

## Example Scenarios

Here are common use cases showing how the plugin adapts to different prompt types:

### Vague Prompt Example

```bash
$ claude "add tests"
```

Claude asks:
```
What would you like to test?
  ○ Unit tests for the authentication service
  ○ Integration tests for the API endpoints
  ○ E2E tests for the checkout flow
  ○ Other (please specify)
```

You select an option, and Claude writes the appropriate tests.

### Clear Prompt Example

```bash
$ claude "Add unit tests for the UserService class in src/services/user.ts, covering the createUser, updateUser, and deleteUser methods with mocked database calls"
```

Claude proceeds immediately without questions and writes the tests.

### Using Bypass Prefixes

If you want to skip evaluation entirely, use the `*` prefix:

```bash
$ claude "* add dark mode"
```

Claude will proceed with the prompt as-is, without evaluation.

<Note>
**Other bypass prefixes:**
- `/` prefix: Slash commands automatically bypass (e.g., `/help`)
- `#` prefix: Memorize commands automatically bypass (e.g., `# remember to use rg over grep`)
</Note>

## Understanding the Flow

Here's what happens behind the scenes:

### For Vague Prompts

1. **Hook intercepts** your prompt
2. **Claude evaluates** clarity using conversation history
3. **Skill invokes** for research and question generation
4. **Research phase**: Claude creates a plan and gathers context from:
   - Your conversation history
   - Codebase files and structure
   - External documentation (if needed)
5. **Question phase**: Claude asks 1-6 grounded questions based on research
6. **Execution**: Claude proceeds with your original request using the clarifications

### For Clear Prompts

1. **Hook intercepts** your prompt
2. **Claude evaluates** clarity using conversation history
3. **Proceeds immediately** - no skill loading, no overhead

## Token Overhead

The plugin is designed to be efficient:

- **Per prompt**: ~189 tokens for evaluation
- **Clear prompts**: Only evaluation overhead (~189 tokens), no skill load
- **Vague prompts**: Evaluation + skill load for comprehensive research
- **30-message session**: ~5.7k tokens (~2.8% of 200k context)

This represents a 31% reduction from previous versions through hook-level evaluation.

## Common Questions

**Will this slow me down?**

Only slightly when it asks questions. Overall, you'll be faster because you get better context upfront and avoid back-and-forth iterations.

**Will I get bombarded with questions?**

No. The plugin rarely intervenes and passes through most prompts. When it does ask, it's limited to 1-6 focused questions.

**Can I customize the behavior?**

The plugin adapts automatically using your conversation history, dynamic research planning, and CLAUDE.md file (if present in your project).

**What if I don't want improvement?**

Use the `*` prefix to bypass evaluation:

```bash
claude "* your prompt here"
```
